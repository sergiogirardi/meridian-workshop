Required Item R1

Reports Module Remediation

The Problem

Filters not fully wired. Approximately 8 logged issues — but the actual number is unknown. The previous vendor's handoff notes acknowledge the module was left in progress.

Our Approach

Full audit against the working parts of the application. We fix all defects found — logged or not. Align Reports to the app's established patterns so it stays fixed.

Fixed-fee covers complete remediation, not a list. If we find more than 8 issues, the price doesn't change.

Required Item R2

Restocking Recommendations

What Operations Gets

A new view: enter a budget ceiling, get back a prioritized list of recommended purchase orders — item, quantity, estimated cost, and why it was flagged.

How It Works

Recommendation logic runs server-side, drawing on existing stock and demand data. Budget ceiling enforced at the recommendation layer — total spend never exceeds the operator's input.

Built on the existing Vue 3 / FastAPI stack. No new infrastructure required.

Required Item R3

Automated Browser Testing

  • End-to-end tests using Playwright — the industry standard for Vue applications
  • Flow inventory defined with IT sign-off — we don't decide what "critical" means unilaterally
  • Covers all critical flows: dashboard, inventory, orders, Reports (post-fix), and Restocking
  • Tests live in the repository, runnable by Meridian IT without vendor involvement
  • Established in Phase 1 — not bolted on at the end
Required Item R4

Architecture Documentation

What's Delivered

A visual architecture diagram with written narrative: component structure, data flow, API surface, filter system, deployment topology. Verified against the actual code.

Why Week 1

The previous vendor's docs are thin. We complete the architecture review at the start — it informs our own work on R1 and R2, and gives IT an immediate deliverable.
