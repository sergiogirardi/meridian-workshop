Known Issues & Findings (R1 Audit)
R1-1 Reports.vue makes API calls with raw axios inline, not via api.js. Inconsistent with every other view — breaks the established pattern and makes testing harder.
R1-2 Reports page has no filter support. All other views respond to the global FilterBar; Reports ignores it entirely. No warehouse/period filtering on quarterly or monthly data.
R1-3 Reports.vue has no i18n — all strings are hardcoded English. Tokyo warehouse staff see no translations on this page.
R1-4 Custom number formatting in Reports.vue (formatNumber, formatMonth) duplicates logic that exists in currency.js and other views. Fragile and untested.
R1-5 Dashboard summary mixes filtered and unfiltered counts: low_stock_items is filtered by warehouse/category, but total_backlog_items counts all backlog regardless of filter. Inconsistent KPI behavior.
R1-6 Backlog on Dashboard shows ALL items regardless of filter bar selections. Backlog.vue applies client-side filtering correctly; Dashboard.vue does not.
R1-7 Order.warehouse and Order.category are Optional in the Pydantic model — used for filtering but not guaranteed present in orders.json. Filter silently returns wrong results for orders missing these fields.
R1-8 purchase_orders.json is empty and the feature is incomplete — PO action buttons visible on Dashboard but the backend data layer is not populated.
